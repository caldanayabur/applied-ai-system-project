# Poisson and Count Models

## Model Eligibility Checklist
- Use only if the dependent variable is integer-valued.
- Use only if the dependent variable is non-negative.
- Use only if the process is a true count-generating process.
- Use only if counts are not truncated into an artificial bounded scale.

## Disqualifying Conditions
- Reject Poisson if the dependent variable is negative.
- Reject Poisson if the dependent variable is not integer-valued.
- Reject Poisson if the dependent variable is bounded but not a count process.
- Reject Poisson if variance is much larger than the mean.
- Reject Poisson if zero inflation is severe and unmodeled.

## Required Diagnostics
- Compute a dependent-variable histogram.
- Compute mean versus variance.
- Compute the proportion of zeros.
- Inspect residual plots after fitting.

## Diagnostic → Action Rules
- If the variable is integer-valued and non-negative, proceed to count-model testing.
- If variance is close to the mean, Poisson is acceptable.
- If variance is much larger than the mean, prefer Negative Binomial.
- If the zero proportion is high, consider a zero-inflated count model.
- If the variable is bounded but not generated by counting events, reject Poisson and route to another family.
- If residual plots show systematic misfit after Poisson, stop and move to an alternative count specification.

## Output Restrictions
- Do not present Poisson results as valid when overdispersion is severe.
- Do not ignore the zero proportion when recommending the model.
- Do not treat bounded non-count outcomes as Poisson data.
